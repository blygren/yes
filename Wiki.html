Introduction

PhySiC is a physics simulation sandbox that allows you to create and interact with various objects in a realistic physics environment. You can spawn different shapes, manipulate their properties, apply forces, and watch how they interact with each other and the environment.

This simulator combines entertaining visual effects with accurate physics, making it both fun to play with and educational. Objects can have faces with emotions, names, and special visual effects, adding character to your simulations.

Getting Started

Basic Controls

When you first open PhySiC, you'll see a blank canvas with a toolbar at the bottom and control panels on the sides:

  • Click anywhere on the screen to spawn an object (in the default spawn mode)
  • Use the bottom toolbar to switch between different tools
  • Access the Settings panel on the right to modify object and world properties
  • Use the Library panel on the left to access shape and color templates
  • Press the spacebar to pause/resume the simulation

Interface Overview

The simulator's interface consists of:

  • Bottom Toolbar: Contains buttons for different interaction tools
  • Settings Panel: Located on the right side, contains all adjustable parameters
  • Library Panel: Located on the left side, contains shapes and color templates
  • Simulation Area: The main canvas where objects interact
  • Inspector Panel: Appears when using the inspect tool on an object

Tools & Modes

PhySiC features several tools that allow you to interact with the simulation in different ways. You can select these tools from the bottom toolbar.

Spawn Tool

The default tool that creates new objects in the simulation.

  • Click anywhere to spawn a single object
  • Click and hold to continuously spawn objects
  • Objects spawn with the currently selected shape, size, and color
  • The spawn rate can be adjusted in settings with the "Spawn Speed" slider

Drag Tool

Allows you to grab and move objects around the simulation.

  • Click and hold on an object to grab it
  • Move your cursor to drag the object around
  • Release to let go of the object
  • When faces are enabled, dragged objects will show a scared expression and may play scared sounds

Attract Tool

Creates an attractive force that pulls objects toward your cursor.

  • Click and hold to activate the attraction force
  • A visual indicator shows the attraction radius
  • Blue particles appear around the cursor when active
  • Adjust attraction strength and radius in the settings

Weld Tool

Creates connections between objects, joining them together.

  • Click on the first object you want to connect
  • The selected object will be highlighted with a green outline
  • Click on a second object to create a weld between them
  • Click the same object again to cancel the selection
  • Toggle "Invisible Welds" in settings to hide the connection lines

Delete Tool

Removes objects from the simulation.

  • Click on an object to delete it
  • Click and drag to delete multiple objects as you move over them
  • Deleted objects will also have any welds connected to them removed

Explode Tool

Creates an explosion that pushes nearby objects away.

  • Click anywhere to create an explosion at that point
  • Objects within the explosion radius will be pushed away
  • A visual shockwave indicates the explosion's area of effect
  • Adjust explosion strength and radius in the settings

Inspect Tool

Provides detailed information about objects in the simulation.

  • Click on any object to view its properties
  • Information appears in a panel showing:
    • Object name (if names are enabled)
    • Shape type
    • Creation time
    • Physical properties (size, position, velocity, etc.)
    • Status information (for objects with faces)
  • Click anywhere else or the close button to close the panel

Object Properties

Shapes

PhySiC supports multiple shapes that you can spawn:

  • Circle: The default shape - smooth rolling movement
  • Square: A perfect square that tumbles and stacks
  • Rectangle: A longer shape that's useful for platforms
  • Triangle: A 3-sided polygon
  • Pentagon: A 5-sided polygon
  • Hexagon: A 6-sided polygon
  • Octagon: An 8-sided polygon
  • Trapezoid: A 4-sided shape with one pair of parallel sides

You can change the shape by clicking the "Shapes" button in the Library panel, which opens a library of available shapes.

Physics Properties

Each object has several physical properties that affect how it behaves:

  • Size: Controlled by the Size slider (unless Random Sizes is enabled)
  • Bounciness: How much an object bounces when it collides
  • Friction: How much an object resists sliding against others
  • Static Friction: The force needed to start an object moving
  • Density: Affects the object's mass and how it responds to forces
  • Rotation: Objects can have random rotation or initial rotation speed
  • Static: Objects can be fixed in place, unaffected by gravity
  • Lifespan: Objects can be set to disappear after a certain time

All of these properties can be adjusted in the Settings panel before spawning objects.

Faces & Emotions

When the "Show Faces" option is enabled, objects will display a simple face that can show different emotions:

Face Features

  • Blinking: Eyes periodically close for a brief moment
  • Looking: Eyes may look at nearby objects with faces
  • Emotions: Faces can show different emotional states:
    • Happy: The default state with a smile
    • Scared: Wide eyes and O-shaped mouth when being dragged
    • Sad: Downturned mouth when damaged or when witnessing a "death"
    • Dead: X eyes and flat mouth when damage threshold is reached
  • Asymmetric Eyes: 5% chance (1 in 20) of having one eye larger than the other

Health System

Objects with faces have a health/damage system:

  • Objects accumulate damage from collisions
  • Circles have a base health threshold of 6.48
  • Rectangles and squares have 30% less health (threshold of 4.54)
  • Once damage exceeds the threshold, the object "dies" (face changes to X eyes)
  • When an object dies, nearby faces become sad temporarily
  • Death triggers a death sound

Invincibility

When "Invincible Faces" is enabled, objects with faces won't take damage and can't die, regardless of collision force.

Settings

Object Settings

These settings control the properties of newly spawned objects:

  • Size: Controls the radius/dimensions of spawned objects (5-50)
  • Random Sizes: When enabled, objects spawn with random sizes between 10-30
  • Random Colors: When enabled, objects spawn with random colors
  • Bounciness: How elastic collisions are (0-1.2)
  • Friction: How much objects resist sliding (0-1)
  • Static Friction: Force needed to start movement (0-1)
  • Density: Affects object mass (0.0001-0.015)
  • Make Static: Objects remain fixed in place
  • Random Rotation: Objects spawn with random rotation angles
  • Rotation Speed: Initial angular velocity (-0.5 to 0.5)
  • Show Faces: Adds faces to objects with emotions and behaviors
  • Invincible Faces: Prevents objects with faces from taking damage
  • Show Names: Gives each object a name from the name pool
  • Manual Name: Lets you specify a custom name
  • Enable Lifespan: Objects disappear after a set time
  • Lifespan: How long objects last before disappearing (1-20 seconds)
  • Object Collisions: When disabled, objects pass through each other
  • Spawn Speed: Rate of continuous spawning when holding mouse button (1-100)

World Settings

These settings affect the simulation environment and physics:

  • Gravity Y: Vertical gravity strength (-1 to 2)
  • Gravity X: Horizontal gravity strength (-1 to 1)
  • Use Accelerometer: Use device tilt to control gravity (mobile devices)
  • Air Friction: How much objects slow down in the air (0-0.25)
  • Time Scale: Simulation speed multiplier (0.1-2.0)
  • Wall Bounciness: How bouncy the boundaries are (0-1.5)
  • Attraction Strength: Force strength for the attract tool (0.1-10)
  • Attraction Radius: Range of the attract tool (50-500)
  • Explosion Strength: Force strength for the explode tool (0.01-0.2)
  • Explosion Radius: Range of the explode tool (50-500)
  • Invisible Welds: Hide the connecting lines between welded objects

Visual Settings

These settings control the appearance of the simulation:

  • Show Grid: Displays a grid background
  • Grid Color: Color picker for the grid lines
  • BG Color: Color picker for the background
  • Show Toolbar: Toggles the visibility of the bottom toolbar
  • Object Colors: Configure up to 9 colors for spawned objects

You can add or remove colors using the Add/Remove buttons, and each color can be customized with a color picker.

Template Management

Templates are saved locally in your browser's storage. To manage your templates:

  • Deleting: Click the "X" button on any custom template to remove it

Statistics & Play Time

PhySiC keeps track of various statistics about your usage:

  • Total Objects: Total number of objects you've spawned
  • Circles: Number of circular objects spawned
  • Polygons: Number of polygon objects spawned
  • Rectangles: Number of rectangular objects spawned
  • Explosions: Number of explosions triggered
  • Objects Deleted: Number of objects manually deleted
  • Objects Cleared: Number of objects removed via Clear Balls
  • Welds Created: Number of welds created between objects
  • Names Given: Number of objects given names
  • Faces Created: Number of objects with faces

The simulator also tracks your total play time across sessions, displayed as hours and minutes.

You can reset your statistics using the "Reset Statistics" button in the settings.

Saving & Loading

PhySiC allows you to save and load simulation states:

Save State

When you click "Save State" in the settings, the simulator will:

  • Create a JSON file containing all current objects and their properties
  • Save all current settings (object properties, world settings, etc.)
  • Save all weld constraints between objects
  • Download the file to your device with a timestamp in the filename

Load State

When you click "Load State" and select a previously saved file:

  • The simulator will first clear all existing objects
  • Restore all settings to the saved values
  • Recreate all objects with their exact properties, positions, and velocities
  • Restore all welds between objects
  • Update the UI to reflect the loaded settings

This feature is useful for saving interesting creations or experiments to revisit later.

Advanced Techniques

PhySiC offers many possibilities for creative physics experiments. Here are some advanced techniques to try:

Perpetual Motion Machines

Create balanced systems that keep objects in motion:

  1. Create a funnel shape using static rectangles
  2. Set wall bounciness to maximum (1.5)
  3. Drop circles with high bounciness (1.2) into the funnel
  4. Watch as the objects bounce continuously with minimal energy loss

Chain Reactions

Design Rube Goldberg-style contraptions:

  1. Create static platforms at various angles
  2. Position balls with different sizes and densities
  3. Use a single delete action or explosion to trigger the chain
  4. Watch the physics-driven chain reaction unfold

Stable Structures

Build stable structures using welding:

  1. Create boxes or polygons for your structure
  2. Use the weld tool to connect them
  3. Create triangular supports for extra stability
  4. Test your structure against forces like gravity or explosions

Newton's Cradle

Create a working Newton's Cradle demonstration:

  1. Create a horizontal line of circles with the same size
  2. Weld each to a static point above it (like a pendulum)
  3. Set bounciness to maximum
  4. Pull one ball to the side and release to watch conservation of momentum in action

Color Sorting Machine

Create a machine that sorts objects by color:

  1. Use static objects to create channels and pathways
  2. Spawn multi-colored objects at the top
  3. Use your creativity to design sorting mechanisms
  4. Collect sorted objects in separate containers at the bottom

Creative Game Modes

While PhySiC is primarily a physics sandbox, you can create your own games and challenges within it:

Target Practice

Create a shooting gallery using the explosion tool:

  1. Set up static "target" objects at various distances
  2. Place dynamic objects on top of them
  3. Use the explosion tool to try and knock all the objects off their platforms
  4. Count how many explosions it takes to clear all targets

Tower Builder

Challenge yourself to build the tallest stable tower:

  1. Spawn objects one at a time
  2. Stack them carefully to build a tower
  3. See how high you can go before it becomes unstable
  4. Use the inspect tool to measure the height of your creation

Domino Rally

Create an elaborate domino setup:

  1. Use rectangle shapes and set them upright
  2. Arrange them in patterns with small gaps between each
  3. Use high static friction to keep them stable
  4. Push the first one to start the chain reaction

Wrecking Ball

Build structures and then destroy them:

  1. Create a structure using static objects
  2. Spawn a large, dense circle
  3. Weld it to a static point above
  4. Pull it back and release to swing it into your structure

Face Survival

A challenge mode using faces:

  1. Enable "Show Faces" but keep "Invincible Faces" off
  2. Create a defined play area with static borders
  3. Spawn a set number of objects with faces
  4. Create obstacles and hazards that might damage them
  5. Try to keep as many faces "alive" as possible

Hidden Features

PhySiC contains several less obvious features you might not have discovered:

Accelerometer Controls

On mobile devices or laptops with motion sensors, you can control gravity using device tilt:

  1. Enable "Use Accelerometer" in the settings
  2. Grant motion sensor permissions when prompted
  3. Tilt your device to change gravity direction
  4. The gravity sliders will update automatically to reflect the current values

Template Search

The color templates library has a search function that lets you quickly find templates:

  • Open the Templates library
  • Type keywords in the search box at the top
  • Results will filter in real-time
  • You can search by template name, category name, or color descriptions

Health System

Objects with faces have a hidden health system with different values based on shape:

  • Circles are the most durable with 6.48 health
  • Squares and rectangles have 30% less health (4.54)
  • Damage is accumulated from collisions based on force
  • Use the Inspect tool to see an object's current health status

Save State File Format

The save state files (.json) can be manually edited with a text editor:

  • Save files contain all object positions, properties, and settings
  • Advanced users can modify these files to create custom scenarios
  • You can share save files with other users

Performance Tips

To get the best performance out of PhySiC, especially on lower-end devices:

Limit Object Count

The single most important factor for performance is the number of objects in the simulation. Try to keep the total under 100 objects for smooth performance on most devices.

Disable Faces and Names

Faces and names require additional rendering and processing. Turn them off for maximum performance when creating complex physical scenarios.

Use Simple Shapes

Circles are the most efficient shape for the physics engine to calculate. Complex polygons require more computational resources.

Limit Special Effects

Special name effects and explosions can impact performance. Use them sparingly in complex scenes.

Slow Down Time

If you notice stuttering, try reducing the Time Scale to 0.5 or 0.75. This gives the physics engine more time to calculate each frame.

Clear Regularly

Use the "Clear Balls" button periodically to prevent accumulation of too many objects.

Feature Performance Impact Alternative
Special Name Effects High Use regular names
Faces with Emotions Medium Disable faces
Many small objects High Fewer, larger objects
Complex polygon shapes Medium Use circles or squares
Many welds Medium Use fewer connections
Continuous explosions High Use attract tool instead

Probabilities & Mechanics

Special Name Probabilities

  • Each object has a 1 in 20 (5%) chance of getting a special name effect
  • If selected for a special effect, there's an equal chance (1/6) of getting any of the 6 special effects
  • This makes the probability of any specific special effect approximately 1 in 132 (0.76%)

Face Mechanics

  • Blinking: Faces blink every 8-20 seconds randomly
  • Damage Threshold: Circles have 6.48 health, squares/rectangles have 4.54 health
  • Looking: 60% chance to look at nearby faces every 2-7 seconds
  • Asymmetric Eyes: 5% chance (1 in 20) of spawning with one eye larger
  • Sadness: Objects stay sad for 5 seconds after witnessing a "death" nearby
  • Fear: Objects show scared expression for 3 seconds when dragged

Troubleshooting

Common Issues

Performance Issues

Problem: Simulation slows down with many objects

Solution: Use the "Clear Balls" button to remove objects, or reduce the number of objects you spawn. Large numbers of objects with faces or special name effects can be particularly demanding.

Audio Issues

Problem: No sound effects playing

Solution: Make sure your device volume is on. PhySiC requires user interaction before audio can play, so click anywhere on the screen first.

Mobile Device Issues

Problem: Accelerometer/tilt controls not working

Solution: Make sure you've granted motion sensor permissions to your browser. Not all devices support accelerometer input.
